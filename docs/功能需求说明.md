# 书签管理扩展 · 功能需求说明

> 本文档梳理当前扩展**已实现**的功能，作为重新设计（claude design）的需求基线。
> 内容如实反映现有行为，包含交互细节、规则边界、数据存储与已知限制，供设计参考。
>
> 适用版本：Manifest V3 · v1.0
> 形态：Edge / Chrome 浏览器扩展，点击工具栏图标后在**整页标签页**中打开书签管理界面。

---

## 1. 产品概述

一个浏览器书签管理扩展，把浏览器原生书签以**美观的整页网格**形式展示，支持浏览、搜索、整理（增/改/删/拖拽）书签，并提供必应每日背景图与可自定义的视觉滤镜。

**核心价值**：替代浏览器原生的下拉式书签栏，提供一个可视化、可整理、可个性化的书签主页。

**数据来源**：完全基于浏览器原生书签（`chrome.bookmarks` API），不维护独立的书签数据库；个性化设置与访问历史存于本地 `localStorage`。

---

## 2. 运行入口

| 入口 | 行为 |
|------|------|
| 点击工具栏扩展图标 | 若已存在书签管理标签页，则切换到该页；否则新开一个标签页打开管理界面 |
| 扩展安装/更新 | 仅记录日志，无引导流程 |

> 注：当前**没有 popup 弹窗**，图标点击直接进入整页界面。

## 3. 功能模块总览

| 编号 | 模块 | 一句话说明 |
|------|------|-----------|
| F1 | 书签网格浏览 | 以网格展示当前文件夹下的书签与子文件夹 |
| F2 | 文件夹层级导航 | 进入子文件夹、返回上级、面包屑路径 |
| F3 | 书签搜索 | 全局关键词搜索书签与文件夹，含路径展示 |
| F4 | 常用/最近访问 | 顶部展示最近打开过的书签图标 |
| F5 | 新建文件夹 | 在当前文件夹下创建子文件夹 |
| F6 | 编辑 | 修改书签的名称/URL，或文件夹名称 |
| F7 | 删除 | 删除书签或整个文件夹（含子项） |
| F8 | 拖拽整理 | 拖拽排序、移入文件夹、移至上级 |
| F9 | 打开书签 | 新标签页打开链接并记录访问 |
| F10 | 必应每日背景 | 自动加载必应每日图片作为背景 |

## 4. 详细功能需求

### F1 · 书签网格浏览

- **需求**：进入界面默认展示「书签栏」（根文件夹 id=`1`）下的内容；书签与子文件夹以网格卡片平铺。
- **卡片内容**：
  - 书签卡片：网站图标 + 标题。图标取自 Google favicon 服务（`s2/favicons`，32px），加载失败回退到本地默认图标。
  - 文件夹卡片：统一文件夹图标 + 文件夹名。
- **特殊卡片**：
  - 「新建文件夹」按钮卡片：任何文件夹层级都存在（含根目录）。
  - 「返回上级」按钮卡片：仅在非根文件夹时出现，排在网格首位。
- **空状态**：当前未实现专门的空文件夹提示（仅展示返回/新建按钮）。

### F2 · 文件夹层级导航

- **进入子文件夹**：点击文件夹卡片，加载其子内容。
- **返回上级**：点击「返回上级」卡片，回到父文件夹。
- **面包屑路径**：
  - 顶部展示从「书签栏」到当前文件夹的完整路径，形如 `书签栏 › A › B`。
  - 路径中除当前项外均可点击，直接跳转到对应层级。
  - 当前项高亮、不可点击。
  - 根节点统一显示为「书签栏」。

### F3 · 书签搜索

- **触发**：顶部搜索框输入 **≥ 2 个字符**时实时搜索；少于 2 字符则隐藏结果。
- **搜索范围**：调用浏览器书签搜索，匹配书签与文件夹。
- **结果展示**：
  - 最多展示 **10 条**。
  - 每条含图标、标题、以及所在路径（`A > B > C`）。
  - 文件夹结果标题以强调色区分。
  - 无匹配时显示「没有找到匹配的书签」。
- **结果点击**：
  - 书签 → 新标签页打开，并记录到访问历史，随后清空搜索框、收起结果。
  - 文件夹 → 导航进入该文件夹。
- **关闭方式**：点击结果区域外、按 `Esc`、或重新聚焦时按规则自动处理。

### F4 · 常用 / 最近访问

- **需求**：顶部「常用网站」区域以小图标形式展示最近打开过的书签，点击可再次打开。
- **记录规则**：
  - 打开书签时记录到本地访问历史（`localStorage`）。
  - 按 id 去重，最新打开的排在最前。
  - 实际存储条数为「显示数量 × 2」，仅保留最近的若干条。
- **展示规则**：
  - 标题固定为「常用网站:」。
  - 展示数量受 F13 设置控制（默认 12）。
  - 无记录时显示「暂无访问记录」。
- **已知行为**：从「常用网站」区域点击图标打开时，当前**不**回写访问历史（不影响排序）。重设计时可考虑统一记录逻辑。

### F5 · 新建文件夹

- **入口**：点击「新建文件夹」卡片，弹出对话框。
- **交互**：输入名称 → 「创建」或回车确认；自动聚焦输入框；点击遮罩或「取消」关闭。
- **校验**：名称为空时输入框抖动提示，不提交。
- **结果**：在当前文件夹下创建子文件夹，Toast 提示并刷新。

### F6 · 编辑书签 / 文件夹

- **入口**：右键卡片 → 弹出上下文菜单 → 选择「编辑」。
- **对话框字段**：
  - 书签：名称 + URL。
  - 文件夹：仅名称。
- **结果**：保存后更新书签数据并刷新当前文件夹；点击遮罩或「取消」关闭。

### F7 · 删除书签 / 文件夹

- **入口**：右键卡片 → 上下文菜单 → 「删除」。
- **确认**：使用浏览器原生 `confirm` 二次确认。
- **规则**：删除文件夹会**连同其所有子项一并删除**（removeTree）。
- **结果**：删除后刷新当前文件夹。

### F8 · 拖拽整理

书签/文件夹卡片均可拖拽（`draggable`），支持三种放置目标：

| 拖到目标 | 行为 |
|----------|------|
| 另一个**书签**上 | 在当前文件夹内重新排序到该位置 |
| **文件夹**卡片上 | 把被拖项移动进该文件夹（置于首位），Toast 提示 |
| 「**返回上级**」卡片上 | 把被拖项移动到上级文件夹（置于首位），Toast 提示 |

- **视觉反馈**：拖拽中源卡片半透明缩小；悬停目标高亮；文件夹/返回目标有专属高亮态。
- **「返回上级」卡片**：自身不可被拖拽，但可作为放置目标。

### F9 · 打开书签

- 点击书签卡片 → 新标签页（`_blank`）打开 URL，并记录访问历史（见 F4）。

### F10 · 必应每日背景

- 页面加载时请求必应每日图片接口，将返回图设为整页背景（铺满、居中）。
- **失败回退**：请求失败时使用纯色背景（`#f0f0f0`）。
- 依赖网络与 `bing.com` 主机权限。

### F11 · 背景滤镜与颜色叠加

在背景图上叠加可调滤镜，所有数值实时生效并持久化。

| 项目 | 范围 | 步长 | 默认值 | 单位 |
|------|------|------|--------|------|
| 亮度 brightness | 0.5 – 2 | 0.05 | 1.05 | — |
| 对比度 contrast | 0.5 – 2 | 0.05 | 1.1 | — |
| 饱和度 saturate | 0 – 2 | 0.05 | 1.2 | — |
| 模糊 blur | 0 – 10 | 0.5 | 0 | px |
| 色调旋转 hue-rotate | 0 – 360 | 10 | 0 | deg |
| 颜色叠加 | 颜色 + 透明度 0 – 1 | 0.05 | 透明度 0 | — |

- **颜色叠加**：在背景上叠加一层指定颜色的半透明蒙层；透明度为 0 时不生效。
- **重置**：「重置滤镜」按钮恢复全部默认值并清除本地存储。
- **加载时机**：滤镜在背景图加载前先写入，避免叠加闪烁。

### F12 · 布局密度设置

- 三档布局，切换即时生效并持久化：
  - **标准**（默认）
  - **紧凑**：更小卡片、更密网格
  - **宽松**：更大卡片、更大间距
- 切换时 Toast 提示当前布局。

### F13 · 最近访问数量设置

- 滑块调节顶部「常用网站」展示数量：范围 **6 – 30**，默认 **12**。
- 调整即时刷新展示并持久化，Toast 提示。

### F14 · 设置面板

- **位置**：固定在右上角。
- **折叠**：点击标题栏展开/收起；**默认收起**。
- **三个标签页**：
  1. **背景滤镜**：F11 全部控件。
  2. **常规设置**：F12 布局选择 + F13 数量滑块。
  3. **关于**：见 F15。

### F15 · 关于页

- 展示：产品名「书签管理器」、版本号、一句话简介、赞助商图片。

### F16 · 操作反馈（Toast）

- 新建文件夹、拖拽移动、布局切换、数量调整等操作成功后，底部居中弹出短暂轻提示。

## 5. 数据与存储

### 5.1 浏览器书签（`chrome.bookmarks`）
书签的增（仅文件夹）、改、删、移动、查询、搜索全部走浏览器原生书签 API，是唯一的书签数据源，与浏览器书签实时同步。

### 5.2 本地存储（`localStorage`）

| Key | 用途 | 默认 |
|-----|------|------|
| `bookmark_visit_history` | 访问历史（最近/常用） | — |
| `bookmark_recent_count` | 最近访问展示数量 | `12` |
| `bookmark_filter_settings` | 滤镜各项数值 | 见 F11 |
| `bookmark_color_settings` | 颜色叠加（颜色 + 透明度） | `#ffffff` / `0` |
| `bookmark_layout_setting` | 布局密度 | `normal` |

> 本地存储为**单设备**、不随账号同步。

| 类型 | 项 | 用途 |
|------|----|------|

| 外部服务 | Google favicon 服务 | 书签网站图标 |
| 外部服务 | 必应每日图片接口 | 背景图 |

## 7. 当前视觉风格现状（供重设计取舍）

- 整体为**玻璃拟态 + 必应大图背景**风格：半透明白面板 + 背景模糊。
- 顶部为合并式工具条（面包屑 + 搜索 + 常用网站同一行）。
- 主色为蓝色系；文件夹卡片带暖色调区分。
- 设置面板为右上角可折叠卡片。

---

## 8. 已知限制（重设计可重点考虑）

1. **无「新增书签」功能**：当前只能新建文件夹，书签依赖浏览器既有数据，不能在界面内手动添加书签。
2. **删除用原生 `confirm`**：体验较生硬，可替换为自定义确认对话框。
3. **搜索受限**：最少 2 字符、最多 10 条结果，无分类/筛选/历史。
4. **访问历史记录不统一**：从「常用网站」区点击打开时未回写历史。
5. **强网络依赖**：背景图、网站图标均依赖外部服务，离线时退化明显。
6. **拖拽排序的位置语义较简单**：移入文件夹/上级时固定置于首位。
7. **无多选 / 批量操作**：增删改均为单项操作。
8. **无书签同步/导入导出、无标签/收藏分组等高级组织能力**。
9. **空文件夹无专门空状态提示**。
10. **移除右键菜单操作方式**

---

## 9. 重设计建议关注点（非强制）

- 顶部信息密度与三类信息（导航/搜索/常用）的优先级与布局。
- 书签卡片信息层级（图标、标题、可选的描述/域名）。
- 整理操作（增删改、拖拽、批量）的可发现性与一致性。
- 个性化（背景、滤镜、布局）的入口与默认值合理性。
- 是否补齐「新增书签」「自定义确认弹窗」「空状态」等缺口。
